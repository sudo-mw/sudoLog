## 배열과 연결 리스트(Linked List)의 차이

> Keyword: 조회, 삽입, 메모리

배열은 메모리 상에 연속적인 공간에 할당된다. 
연결 리스트는 연속적인 공간에 할당되지 않고 노드 안에 다음 노드의 메모리 주소를 저장한다. 

배열은 임의의 원소 접근에 O(1)만큼 사용하고, 연결 리스트는 순차적으로 접근해야하기 때문에 O(n)이 필요하다. 

배열은 중간에 삽입 시 뒤에 있는 원소들의 위치를 바꿔야하기 때문에 O(n)이 필요하고, 연결 리스트는 O(1)이 필요하다. 

## 캐시 

데이터를 빠르게 처리하기 위해서는 CPU가 데이터를 캐시 메모리에 보유하고 있어야한다. 만약 찾으려는 데이터가 캐시에 없다면(Cache Miss), CPU는 상대적으로 속도가 느린 RAM까지 가서 데이터를 가져온다. 

CPU는 효율성을 위해 데이터 하나만 가져오는 것이 아니라 인접한 메모리를 통째로 읽어와 캐시를 채운다. (캐시 필)

이 때, 배열은 데이터가 메모리 상에 연속적으로 위치하는 공간 지역성(Spatial Locality)을 가지기 때문에 한 번의 캐시 필만으로도 이후에 처리할 데이터들이 함께 캐시로 들어온다. 

반면, 리스트는 노드들이 힙 메모리 곳곳에 불연속적으로 흩어져잇다. 캐시 라인을 채워봤자 정작 필요한 다음 노드는 포함되지 않을 확률이 높고, 결과적으로 빈번한 Cache Miss와 RAM 접근을 유발하여 성능이 저하됩니다.
